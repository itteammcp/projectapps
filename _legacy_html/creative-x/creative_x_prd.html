1. Pendahuluan & Latar Belakang

1.1 Latar Belakang

PT Multicrane Perkasa sebagai salah satu perusahaan distribusi dan layanan alat berat terkemuka di Indonesia membutuhkan sistem terintegrasi yang mampu menyelaraskan operasional lapangan dan administrasi keuangan. Sebelumnya, terdapat kendala akibat fragmentasi data antara pelacakan status unit alat berat di lapangan (hours meter, servis, garansi) dengan administrasi keuangan (penagihan piutang, kasbon karyawan, komisi agen).

Data yang tersebar di berbagai platform pihak ketiga, salah satunya Accurate Online (AOL) untuk pencatatan akuntansi dan sistem HRIS untuk kepegawaian, memicu redundansi kerja dan keterlambatan pengambilan keputusan. Oleh karena itu, platform Creative-X dikembangkan sebagai solusi enterprise terpusat (ERP) guna menjembatani sistem transaksional akuntansi eksternal, manajemen data kepegawaian, dan operasional unit alat berat secara real-time.

1.2 Visi & Tujuan Creative-X

Creative-X dirancang dengan visi menjadi *"Single Source of Truth"* (Satu Sumber Kebenaran Data) bagi seluruh operasional perusahaan.

Tujuan utama aplikasi meliputi:

1. Mengurangi Silo Data: Mengintegrasikan data keuangan dari Accurate Online dengan data operasional lapangan.

2. Otomatisasi Alur Kerja (Workflow Automation): Mengurangi proses manual melalui otomatisasi pengingat piutang (*AR Reminders*) dan persetujuan bertingkat (*Approval Engine*).

3. Efisiensi Lapangan: Memfasilitasi mekanik dan operator dalam pelaporan unit alat berat secara mobile dan terstandarisasi.

4. Keamanan & Skalabilitas Tinggi: Menerapkan arsitektur multi-database, manajemen otorisasi ketat berbasis peran (*RBAC*), dan sistem pencatatan audit trail menyeluruh.

  • Tim Operasional Fleet (Ares Staff & Managers): Memantau performa, utilisasi, dan riwayat pemeliharaan alat berat.

2.1 Pendekatan Multi-Database

Untuk menjamin keamanan data global dan memberikan performa optimal, sistem membagi tabel ke dalam tiga domain utama:

1. Settings Database (settingsDB):

Menyimpan seluruh konfigurasi global sistem dengan prefix tabel t_cfg_. Domain ini dirancang independen agar konfigurasi organisasi dapat digunakan kembali jika aplikasi dikembangkan menjadi multi-tenant.

2. Core Database (coreDB):

Menyimpan data transaksi operasional inti perusahaan, data pengguna (t_core_users), relasi hak akses pengguna (t_core_user_roles), alat berat (ares schema), masterdata pelanggan (t_md_*), dan data persetujuan (t_core_approvals).

3. Application Logs (application_logs schema):

Menyimpan log audit trail aktivitas pengguna (t_logs_users), catatan sinkronisasi eksternal (t_logs_system_syncs), log pengiriman notifikasi piutang (t_logs_ar_notifications), dan notifikasi sistem (t_logs_system_notifications).

2.2 Kebijakan Berbagi Database (Co-existence: Creative-X & Talos)

Selama masa pengembangan, sistem Creative-X berjalan berdampingan dengan platform sekunder Talos pada satu instance Supabase PostgreSQL yang sama.

Spesifikasi Detail Deskripsi & Fitur
Deskripsi
Fitur Utama
  • Pola Isolasi: Data transaksional diisolasi menggunakan schema terpisah (ares, moneta vs talos schema).
  • Pola Bersama (Shared Schema): Kedua aplikasi berbagi data global pada schema public terutama untuk tabel pengguna (t_core_users), roles, dan settings global. Pemuatan menu sidebar disaring berdasarkan atribut scope (app_scope = 'creative-x' atau 'all') untuk menghindari bentrokan menu di antarmuka masing-masing aplikasi.

1. Submodul: Autentikasi Keamanan (JWT Auth)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Mengamankan semua komunikasi API menggunakan JSON Web Token (JWT).
Fitur Utama
  • Login menggunakan kombinasi Username/Email dan Password terenkripsi (Bcrypt).
  • Pengeluaran Access Token (umur pendek) dan Refresh Token (umur panjang) yang aman di HTTP-only cookie.
  • Validasi token di backend menggunakan middleware (JWTMiddleware) untuk memverifikasi hak akses.
  • Fitur keluar logis (*Logout*) untuk membatalkan keabsahan sesi token.

2. Submodul: Profiling & Struktur Pengguna (Users Directory)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Mengelola profil data karyawan secara terperinci lengkap dengan hierarki struktur organisasi tempat mereka bekerja.
Fitur Utama
  • Informasi Data Kaya: Nama lengkap, email, nomor HP, tanggal lahir, tanggal bergabung, NIK (Nomor Induk Karyawan), NPWP, Level Otoritas (Staff, Supervisor, Manager, Director), dan Jabatan pekerjaan (Job Title).
  • Penugasan Organisasi: Penugasan terikat pada satu Region (Wilayah), Division (Divisi), dan Department (Departemen) guna membatasi cakupan wilayah operasional dan visualisasi data transaksional pengguna di antarmuka.
  • Hierarki Atasan Langsung (Supervisor ID): Mendefinisikan struktur pelaporan langsung (self-referencing user relation) yang akan digunakan secara otomatis oleh *Approvals Engine* untuk persetujuan berjenjang.
  • Unggah Avatar: Pengguna dapat mengunggah foto profil yang divalidasi format dan ukurannya di backend, lalu disimpan di penyimpanan lokal /uploads/avatars/.

3. Submodul: Integrasi Multi-Identitas Pengguna (Account Linking)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Menghubungkan akun internal pengguna Creative-X dengan identitas unik mereka di sistem eksternal.

  • Accurate Online Link: Menautkan aol_user_id atau aol_supplier_id (untuk agen eksternal) untuk memproses pencatatan transaksi kasbon dan klaim biaya operasional secara langsung ke buku besar Accurate.

5. Submodul: Organizational Structures (Divisions & Departments)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Mengelola pemetaan struktur departemen kerja di bawah divisi bisnis tingkat atas.
Fitur Utama

  • Departemen: CRUD departemen kerja (seperti *ICT Development*, *Account Receivable*, *Parts Logistics*) dengan penetapan divisi induk (*Division ID*) melalui dropdown dinamis di frontend.

6. Submodul: Role-Based Access Control (RBAC Engine)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Mengelola hak akses sistem secara atomik dan dinamis untuk meminimalkan risiko keamanan data.
Fitur Utama
  • Permissions Registry: Mendaftarkan kode hak akses atomik baru ke database (contoh: view-regions, moneta.ar-inflow.view, ares.equipments.create).
  • Role Registry: Membuat peran khusus (seperti *Super Admin*, *AR Supervisor*, *Mechanic*, *Operator*).
  • Assign Permissions: Dialog khusus (RolePermissionsDialog) untuk menautkan/mencentang hak akses mana saja yang boleh digunakan oleh suatu Role secara instan ke tabel relasi t_cfg_role_permissions.

7. Submodul: Dynamic Menu Management (Sidebar Layout)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Mengatur struktur navigasi sidebar utama di frontend secara dinamis berdasarkan data menu yang disimpan di database.
Fitur Utama
  • Hierarki Menu: Mendukung menu bersarang (parent dan sub-menu) yang dirender otomatis dengan simbol indentasi (↳) di tabel pengaturan.
  • Visual Icon & Route: Mengatur nama ikon Lucide (seperti LayoutDashboard, Wrench, Settings) dan rute URL tujuan navigasi (seperti /moneta/ar-inflow).
  • Required Permission: Menetapkan kode hak akses wajib agar menu tersebut muncul di sidebar pengguna.
  • Scope filtering: Memilah menu spesifik Creative-X vs Talos.

8. Submodul: External Integration Settings (AOL Gateway)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Pusat kendali kredensial penghubung aplikasi dengan Accurate Online (AOL) API.
Fitur Utama
  • OAuth2 Config: Mengelola parameters rahasia Client ID, Client Secret, Token Akses (access_token), dan Token Pembaruan (refresh_token).
  • Automated Reminders Toggle: Sakelar global untuk mengaktifkan/menonaktifkan pengiriman email penagihan piutang otomatis ke pelanggan.
  • Sync Scheduler: Penjadwalan jam sinkronisasi data harian otomatis (format HH:mm).
  • AOL Database Connector: Menghubungkan ID database Accurate Online (aol_database_id).

1. Submodul: AR Inflow Management (Piutang Pelanggan)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Menampilkan dan melacak status seluruh faktur penjualan (*Invoices*) pelanggan PT Multicrane Perkasa yang ditarik secara real-time/terjadwal dari Accurate Online.
Fitur Utama
  • Dasbor visualisasi ringkasan piutang: Total Invoice, Jumlah Invoice Terbayar (*Paid*), Belum Terbayar (*Unpaid*), dan Terbayar Sebagian (*Partial*).
  • Filter pencarian faktur berdasarkan nomor faktur, rentang tanggal, status pembayaran, dan nama pelanggan.
  • Penghitungan umur piutang (*Aging Schedule*) untuk menganalisis risiko penagihan.

2. Submodul: AR Invoice Special Treatments

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Memberikan fleksibilitas khusus pada penagihan piutang pelanggan strategis tanpa mengacaukan pembukuan utama di Accurate Online.
Fitur Utama
  • Special Customer Grace Period: Menetapkan hari kelonggaran tambahan secara global untuk pelanggan tertentu (contoh: tambahan grace_period_days = 14 hari) sebelum notifikasi peringatan mulai dikirimkan.
  • Special Due Date Treatment: Menetapkan tanggal jatuh tempo khusus secara sepihak untuk nomor faktur tertentu di tabel t_ar_invoice_special_due_dates.

3. Submodul: Automated AR Reminders (Notifikasi Piutang)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Sistem pengingat otomatis untuk mengirimkan email tagihan kepada pelanggan sebelum dan pada saat jatuh tempo guna mempercepat arus kas masuk (*inflow*).
Fitur Utama
  • Configurable Milestones: Super Admin dapat mengatur interval hari pengiriman (contoh: pengingat ke-1 dikirim H-7 jatuh tempo, pengingat ke-2 dikirim H-3, pengingat ke-3 dikirim pas hari H jatuh tempo).
  • Rich Email Templates: Menghasilkan draf email penagihan dinamis lengkap dengan daftar tagihan, total outstanding, dan tautan pembayaran.
  • Notification Logs: Menyimpan log pengiriman notifikasi (application_logs.t_logs_ar_notifications) untuk melacak status terkirim (*sent*) atau gagal (*failed*) beserta pesan kesalahan dari server SMTP.

4. Submodul: Pinjaman Sementara (PS / Cash Advance)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Memfasilitasi karyawan dalam pengajuan dana tunai kasbon operasional kantor sebelum beraktivitas di lapangan (misal: perjalanan dinas, pembelian suku cadang darurat).
Fitur Utama
  • Form Pengajuan Kasbon: Input jumlah nominal dana, tujuan penggunaan (*purpose*), rekening bank tujuan transfer (Nama Bank, Nomor Rekening), dan estimasi tanggal penyelesaian kasbon.
  • Integrasi Approvals: Pengajuan otomatis masuk ke alur persetujuan bertingkat.
  • Accurate Online Bill Creation: Setelah disetujui penuh oleh jajaran manajemen, status kasbon berubah menjadi disbursed. Sistem secara otomatis membuat draft dokumen transaksi pembelian (*Purchase Bill*) di Accurate Online untuk proses pengeluaran dana akuntansi, lengkap dengan penomoran berurutan otomatis (contoh: PS26.299) berdasarkan transaksi live terakhir di AOL.

5. Submodul: Pinjaman Sementara Settlement (Penyelesaian Kasbon)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Proses pertanggungjawaban karyawan atas penggunaan dana kasbon setelah aktivitas lapangan selesai.
Fitur Utama
  • Expense Breakdown: Karyawan menginput pos-pos pengeluaran riil berdasarkan kategori resmi (contoh: hotel, transportasi, makan) lengkap dengan deskripsi dan jumlah nominal.
  • Attachment Proofs: Mengunggah file bukti kuitansi/nota belanja digital ke cloud server.
  • Refund / Overspent Calculation: Sistem menghitung selisih otomatis. Jika sisa dana kasbon masih ada, sistem meminta karyawan mengunggah bukti transfer pengembalian dana (*cash return*). Jika pengeluaran melebihi nominal kasbon, sistem memproses klaim ganti rugi (*reimbursement*).
  • Persetujuan Settlement: Alur persetujuan bertingkat atas rincian pengeluaran oleh Finance dan Supervisor. Setelah disetujui, kasbon ditandai selesai (settled).

6. Submodul: Commission Calculator (Kalkulator Komisi)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi Modul pembantu bagi sales dan agen eksternal untuk menghitung persentase komisi atas penjualan unit alat berat yang berhasil ditagih piutangnya (*AR cleared*).
Fitur Utama Formula komisi yang disesuaikan dengan profit margin dan tipe unit mesin yang terjual.

3. Kebutuhan Non-Fungsional (Non-Functional Requirements)

Untuk menjamin kenyamanan dan keandalan operasional, sistem Creative-X harus memenuhi parameter performa dan keamanan berikut:

4.1 Performa & Kecepatan Respon (Performance & SLA)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi
Fitur Utama
  • Response Time: Endpoint API backend harus merespon permintaan dalam waktu kurang dari 200ms untuk operasional normal, dan maksimal 1500ms untuk proses integrasi data eksternal (AOL sync).
  • Caching: Menerapkan mekanisme caching data statis (seperti data provinsi, kota, dan menu navigasi) guna memperingan beban transaksi database PostgreSQL.
  • Concurrent Users: Sistem dirancang untuk mampu melayani minimal 200 pengguna aktif secara bersamaan tanpa degradasi performa yang berarti.

4.2 Ketersediaan & Skalabilitas (Availability & Reliability)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi
Fitur Utama
  • SLA Uptime: Menjamin tingkat ketersediaan sistem (*Uptime*) minimal sebesar 99.9% setiap bulannya.
  • Soft Delete: Semua tabel data transaksional wajib menerapkan konsep GORM Soft Delete (deleted_at timestamp) untuk menghindari hilangnya data secara permanen akibat kesalahan manusia.
  • Database Backup: Melakukan pencadangan database (*database backup*) secara otomatis setiap hari pada pukul 01:00 WIB ke penyimpanan awan terenkripsi.

4.3 Keamanan Informasi & Privasi (Security & Compliance)

Spesifikasi Detail Deskripsi & Fitur
Deskripsi
Fitur Utama
  • Transport Encryption: Seluruh lalu lintas data antara klien Next.js dan backend API Go wajib dilindungi enkripsi HTTPS (SSL/TLS 1.3).
  • Data Masking: Semua kata sandi dalam database wajib di-hash menggunakan algoritma Bcrypt dengan cost factor minimal 10. API dilarang mengembalikan password hash dalam payload JSON.
  • Sensitive Credentials: Kredensial eksternal seperti Client Secret AOL wajib disimpan dalam keadaan terenkripsi di tingkat database menggunakan kunci enkripsi AES-256 yang aman di environment server.
