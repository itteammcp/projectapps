01

Ringkasan Eksekutif

Warehouse Management System (WMS) ini dirancang untuk mengelola operasional gudang secara menyeluruh — mulai dari penerimaan barang, penyimpanan berbasis racking & bin, pengelolaan barcode serialized, assemby/disassembly dengan Bill of Materials (BOM), hingga sinkronisasi dua arah dengan Accurate Online sebagai sistem akuntansi utama.

Sistem ini dibangun untuk mendukung kebutuhan orang gudang, admin spare part, dan parts analyst dengan antarmuka yang intuitif, dukungan multi-warehouse, serta visualisasi tata letak gudang berbasis 2D. Seluruh transaksi stok dan mutasi barang yang terjadi di WMS akan otomatis tersinkronisasi ke Accurate Online untuk menjaga konsistensi data keuangan dan inventori.

ℹ
Dokumen ini merupakan Product Requirements Document (PRD) versi awal yang akan diperbaharui secara iteratif seiring berjalannya proses discovery dan development.
02

Tujuan & Latar Belakang

Latar Belakang

Pengelolaan gudang yang dilakukan secara manual atau dengan spreadsheet rentan terhadap human error, ketidakakuratan stok, dan sulitnya penelusuran riwayat barang. Kebutuhan akan sistem yang mampu melacak setiap unit barang secara serial (serialized tracking), mengelola gudang dengan struktur bertingkat (pallet, dus, satuan), serta terintegrasi langsung dengan software akuntansi menjadi prioritas utama.

Tujuan Bisnis

  • Meningkatkan akurasi data stok hingga ≥99% melalui sistem barcode serialized.
  • Mempercepat proses penerimaan, pengeluaran, dan perpindahan barang antar gudang.
  • Mengurangi waktu pelaksanaan stock opname hingga 60% dibandingkan proses manual.
  • Memberikan visibilitas penuh atas lokasi barang (rack, bin, level) secara real-time.
  • Mengeliminasi entri data ganda antara sistem gudang dan Accurate Online.
  • Menyediakan data analitik untuk mendukung keputusan pengadaan dan perencanaan stok.

Tujuan Produk

  • Membangun WMS berbasis web yang dapat diakses dari desktop maupun perangkat mobile/tablet.
  • Menyediakan API yang robust untuk integrasi dua arah dengan Accurate Online.
  • Mendukung operasional multi-warehouse dalam satu platform terpadu.
  • Memastikan performa sistem tetap optimal bahkan dengan volume transaksi tinggi.

Fitur Utama

5.1 Sinkronisasi dengan Accurate Online

Integrasi dua arah antara WMS dan Accurate Online memastikan konsistensi data antara operasional gudang dan catatan akuntansi/keuangan perusahaan.

🔄
Sinkronisasi Real-time

Setiap transaksi penerimaan, pengeluaran, dan penyesuaian stok di WMS otomatis dikirim ke Accurate Online menggunakan webhook atau polling interval.

📦
Sync Master Barang

Data item/produk dari Accurate Online (kode, nama, satuan, harga pokok) otomatis disinkronisasi sebagai master barang di WMS.

📋
Sync Dokumen Transaksi

Purchase Order, Sales Order, dan dokumen gudang (GRN, Delivery Order) sinkron dua arah. Nomor dokumen Accurate dapat digunakan sebagai referensi.

⚠️
Conflict Resolution

Mekanisme penanganan konflik data saat sinkronisasi gagal, termasuk antrian retry otomatis, log error, dan notifikasi admin.

⚡
Catatan Teknis: Integrasi menggunakan Accurate Online REST API v1. Autentikasi menggunakan OAuth2 dengan token yang diperbarui secara otomatis. Rate limit API Accurate Online perlu diperhitungkan dalam desain antrian.

Data yang Disinkronisasi

Data Arah Frekuensi Keterangan
Master Barang Accurate → WMS On-demand + scheduler Kode, nama, satuan, kategori, HPP
Stok Masuk (GRN) WMS → Accurate Real-time Jurnal penerimaan barang
Stok Keluar (DO) WMS → Accurate Real-time Jurnal pengeluaran barang
Penyesuaian Stok WMS → Accurate Real-time Hasil stock opname atau koreksi manual
Assembly/Disassembly WMS → Accurate Real-time Konversi bahan menjadi produk jadi
Data Gudang WMS ↔ Accurate On-demand Mapping gudang WMS ↔ lokasi Accurate

5.2 Barcode Barang — Konsep Serialized

Setiap unit barang diidentifikasi dengan nomor serial unik yang dikodekan dalam barcode (1D/2D). Pendekatan ini memungkinkan penelusuran riwayat per unit dari penerimaan hingga pengeluaran.

🏷️
Penerbitan Serial Number

Serial number dapat digenerate otomatis oleh sistem (format konfigurabel) atau diinput manual sesuai serial dari vendor/produsen.

📱
Scan via Mobile/Scanner

Mendukung scan menggunakan kamera smartphone, barcode scanner USB/Bluetooth, maupun barcode scanner industri (Zebra, Honeywell).

🔍
Traceability Lengkap

Riwayat lengkap setiap serial: dari PO masuk → lokasi bin → perpindahan → assembly/disassembly → pengeluaran atau disposal.

🖨️
Cetak Label Barcode

Cetak label barcode secara massal atau satuan, mendukung format QR Code dan Code 128. Template label dapat dikustomisasi.

Format Serial Number

Komponen Contoh Keterangan
Kode Gudang WH01 Identifikasi gudang asal
Kode Barang SP-FILTER Kode item dari master barang
Tanggal 20250510 Tanggal penerimaan (YYYYMMDD)
Sequence 0001 Nomor urut reset per hari/per item
Contoh Lengkap WH01-SP-FILTER-20250510-0001 Barcode: Code 128 atau QR Code

5.3 Layouting Gudang dengan Desain 2D

Representasi visual tata letak gudang memungkinkan pengguna memahami posisi fisik barang secara intuitif dan melakukan perencanaan penyimpanan yang efisien.

🗺️
Canvas 2D Interaktif

Editor drag-and-drop berbasis canvas (SVG/Canvas API) untuk mendefinisikan denah gudang: zona, lorong, rack, dan area khusus.

🎨
Zona & Area

Pembagian gudang ke zona (receiving, storage, dispatch, quarantine) dengan kode warna berbeda untuk identifikasi cepat.

📍
Heatmap Stok

Visualisasi kepadatan stok per lokasi dalam tampilan heatmap — membantu identifikasi area overstock atau kosong dengan cepat.

🔎
Klik untuk Detail

Klik pada rack/bin di layout 2D langsung menampilkan informasi stok, daftar item, dan riwayat transaksi lokasi tersebut.

5.4 Peletakan Barang Multi-Level

Sistem mendukung hierarki penyimpanan barang yang fleksibel — dari level palet paling besar hingga satuan terkecil, serta level kustom yang dapat dikonfigurasi sesuai kebutuhan operasional.

Hierarki Level (Default)

Level Deskripsi Contoh Kapasitas Keterangan
Level 1 — Palet Unit paling besar, biasanya di atas palet kayu 1 palet = N dus Barcode pada palet
Level 2 — Dus / Karton Unit pengepakan menengah 1 dus = N satuan Barcode pada dus
Level 3 — Satuan Unit terkecil / individual item 1 unit = serialized Barcode serial per unit
Level Kustom Level tambahan sesuai kebutuhan Konfigurabel Contoh: bundle, set, kit
ℹ
Setiap level memiliki barcode/label tersendiri. Saat scan level tinggi (palet/dus), sistem dapat langsung memproses semua unit di dalamnya secara batch.

5.5 Racking & Bin Management

🗄️
Struktur Lokasi Bertingkat

Hirarki: Gudang → Zona → Aisle → Rack → Level Rak → Bin. Setiap bin memiliki kode unik dan kapasitas yang dapat dibatasi.

📏
Kapasitas & Dimensi

Konfigurasi kapasitas per bin (volume, berat, atau jumlah unit). Sistem memberikan peringatan saat kapasitas terlampaui.

🎯
Putaway Suggestion

Sistem merekomendasikan lokasi bin optimal untuk penyimpanan barang baru berdasarkan aturan yang dikonfigurasi (FIFO, kategori, frekuensi).

🔒
Bin Blocking

Kemampuan untuk memblokir bin tertentu dari aktivitas (sedang maintenance, karantina barang, atau reserved untuk barang khusus).

Format Kode Lokasi

Contoh: WH01-A-R03-L2-B05 → Gudang 1, Zona A, Rack 03, Level 2, Bin 05

5.6 Assembly & Disassembly + BOM

Fitur ini memungkinkan penggabungan beberapa komponen menjadi satu produk jadi (assembly) atau pemecahan produk menjadi komponen penyusunnya (disassembly).

🔧
Manajemen BOM

Definisi Bill of Materials multi-level: satu produk jadi dapat terdiri dari komponen yang masing-masing memiliki BOM tersendiri (nested BOM).

⚙️
Proses Assembly

Penggabungan komponen sesuai BOM. Sistem otomatis mengurangi stok komponen dan menambah stok produk jadi. Serial number produk jadi digenerate otomatis.

🔨
Proses Disassembly

Pemecahan produk jadi kembali ke komponen. Stok produk berkurang dan stok komponen bertambah sesuai BOM. Kondisi komponen dapat diinput (baik/rusak).

📄
Work Order Assembly

Proses assembly/disassembly menggunakan Work Order dengan status tracking (pending, in-progress, completed). Mendukung partial assembly.

Contoh Alur Assembly

Langkah Aksi Efek Stok
1 Buat Work Order Assembly untuk produk "Kit Filter A" —
2 Sistem mengecek ketersediaan komponen sesuai BOM Reserve stok komponen
3 Petugas gudang scan & konfirmasi komponen yang digunakan —
4 Selesaikan Work Order, input jumlah produk jadi Kurangi stok komponen, tambah stok produk jadi
5 Sistem generate serial number untuk produk jadi Produk jadi tersedia di bin yang ditentukan
6 Otomatis sync ke Accurate Online (jurnal konversi) HPP dihitung ulang

5.7 Stock Opname per Gudang

📊
Cycle Count & Full Count

Mendukung dua mode: full stock opname (semua item gudang) dan cycle count (opname periodik per zona/kategori tanpa menghentikan operasi).

👥
Multi-Petugas

Penugasan area/bin berbeda ke petugas berbeda secara bersamaan. Real-time progress tracking per petugas.

⚖️
Rekonsiliasi Otomatis

Perbandingan otomatis antara stok sistem vs hasil hitungan fisik. Selisih (variance) ditampilkan per item dengan highlight untuk perbedaan signifikan.

✅
Approval & Adjustment

Hasil opname memerlukan persetujuan supervisor sebelum penyesuaian stok dijalankan dan disinkronkan ke Accurate Online.

5.8 Multi Warehouse

Sistem mendukung pengelolaan beberapa gudang dalam satu platform dengan visibilitas terpusat dan kontrol akses berbasis gudang.

  • Setiap gudang memiliki konfigurasi tersendiri: layout, zona, rack, bin, dan aturan penyimpanan.
  • Transfer barang antar gudang dengan dokumen Transfer Order yang memerlukan konfirmasi di kedua sisi (pengirim & penerima).
  • Dashboard agregat yang menampilkan total stok lintas gudang sekaligus view per gudang.
  • Laporan stok, mutasi, dan KPI dapat difilter per gudang atau lintas gudang.
  • Kontrol akses berbasis gudang: pengguna dapat dibatasi hanya mengakses gudang tertentu.
  • Mapping gudang WMS ke kode lokasi/gudang di Accurate Online.

5.9 Reporting & Analitik

📈
Dashboard Operasional

Real-time KPI: total item, nilai stok, aktivitas inbound/outbound hari ini, item dengan stok rendah, dan pending work orders.

🗂️
Laporan Inventori

Stok per gudang/bin, mutasi barang, aging stok, slow-moving items, dan laporan nilai inventori berdasarkan metode FIFO/Average.

🔍
Laporan Transaksi

Riwayat penerimaan, pengeluaran, transfer, assembly/disassembly, dan stock opname dengan filter tanggal, gudang, barang, dan pengguna.

⬇️
Ekspor Laporan

Ekspor ke format Excel (XLSX), CSV, dan PDF. Penjadwalan laporan otomatis dengan pengiriman via email.

06

Fitur Tambahan (WMS Dasar)

📥
Goods Receipt (Penerimaan Barang)

Proses penerimaan dari Purchase Order atau tanpa PO (direct receipt). Verifikasi quantity dan kondisi barang, cetak label barcode otomatis.

📤
Goods Issue (Pengeluaran Barang)

Proses pengeluaran berdasarkan Sales Order, permintaan internal, atau transfer. Mendukung strategi FIFO, FEFO, dan LIFO.

↔️
Stock Transfer

Pemindahan stok antar bin, antar zona, maupun antar gudang berbeda. Dilengkapi dokumen Transfer Order dan track status.

✏️
Stock Adjustment

Penyesuaian stok manual dengan alasan (damage, loss, found, dll). Memerlukan persetujuan dan otomatis mencatat ke audit log.

🗃️
Master Data Management

Manajemen master item, kategori, satuan, supplier, customer, dan gudang. Import bulk via Excel template.

🔔
Alert & Notifikasi

Peringatan stok minimum (reorder point), masa expired barang, kapasitas bin, dan pending approval yang belum diselesaikan.

🧾
Audit Log

Pencatatan lengkap semua aktivitas pengguna: who, what, when, before value, after value untuk keperluan audit dan troubleshooting.

📱
Mobile-Friendly Interface

UI responsif untuk akses dari tablet dan smartphone. Mode khusus operasional gudang dengan tombol besar untuk kemudahan penggunaan dengan sarung tangan.

📋
Picking & Packing

Pick list otomatis berdasarkan order, optimasi rute picking (wave picking), konfirmasi packing dengan scan barcode.

Non-Functional Requirements

<200ms
Response Time API
95th percentile untuk endpoint CRUD standar pada kondisi load normal.
99.5%
Uptime SLA
Ketersediaan sistem minimal 99.5% (≤3.65 jam downtime/bulan) di luar maintenance window.
1000+
Concurrent Users
Sistem harus mampu menangani minimal 1000 pengguna aktif bersamaan tanpa degradasi performa signifikan.
AES-256
Enkripsi Data
Data sensitif terenkripsi at-rest (AES-256) dan in-transit (TLS 1.3). Token OAuth disimpan terenkripsi.
RBAC
Access Control
Role-Based Access Control granular per modul dan per gudang. Multi-factor authentication untuk super admin.
WCAG 2.1
Aksesibilitas
UI memenuhi standar aksesibilitas WCAG 2.1 Level AA untuk mendukung pengguna dengan kebutuhan khusus.
Daily
Backup & Recovery
Backup otomatis harian dengan RTO ≤4 jam dan RPO ≤1 jam. Backup disimpan di minimum 2 lokasi berbeda.
iOS/Android
Mobile Compatibility
Antarmuka responsif dan fungsional di browser mobile iOS (Safari) dan Android (Chrome) versi terbaru.

Risiko & Mitigasi

Risiko Probabilitas Dampak Strategi Mitigasi
API Accurate Online berubah / tidak stabil Sedang Tinggi Buat abstraction layer untuk integrasi. Monitor changelog API Accurate. Sediakan mode offline dengan sync manual.
Kompleksitas data migration dari sistem lama Tinggi Sedang Audit data awal sejak Fase 0. Buat script migration yang dapat diuji berulang. Lakukan dry-run migration sebelum cutover.
Resistensi user terhadap sistem baru Sedang Sedang Libatkan user dalam UAT. Buat dokumentasi & video tutorial. Sediakan support hotline selama periode hypercare.
Performa sistem saat volume transaksi tinggi Rendah Tinggi Load testing sejak Fase 2. Implementasi caching Redis. Database indexing yang optimal. Horizontal scaling dengan Kubernetes.
Scope creep dari permintaan fitur tambahan Tinggi Sedang Strict change management process. Fitur baru masuk backlog dan dievaluasi per sprint. Freeze scope 4 minggu sebelum go-live.
Ketersediaan infrastruktur / cloud outage Rendah Tinggi Multi-zone deployment. Automated failover. RTO/RPO yang terdefinisi dengan prosedur DR yang diuji secara berkala.

Asumsi & Dependensi

Asumsi

  • Perusahaan telah memiliki akun Accurate Online aktif dengan paket yang mendukung akses API.
  • Infrastruktur server (cloud/on-premise) tersedia sebelum dimulainya Fase 1.
  • Setiap gudang telah dilengkapi jaringan internet yang stabil untuk operasional real-time.
  • Petugas gudang akan dilengkapi dengan perangkat tablet atau barcode scanner yang kompatibel.
  • Tersedia data master barang yang clean untuk proses migrasi awal.
  • Tim IT internal atau vendor IT dapat mengelola deployment dan pemeliharaan infrastruktur.

  • Accurate Online API: Ketersediaan dan stabilitas API v1 dari Accurate Online.
  • Printer Label: Printer barcode kompatibel (Zebra ZPL atau TSPL) untuk cetak label.
  • Barcode Scanner: Scanner yang mendukung HID (keyboard emulation) atau Bluetooth SPP.
  • Cloud Provider: AWS / GCP / Azure untuk hosting production environment.
  • CDN: Untuk distribusi aset frontend dan optimasi performa global.
